# BC-AI 桥接：动态事件字段生成规则

本文档描述 `BellumCivileAIInfluencePatch` 在 BC 游戏事件触发时，向 AIInfluence `dynamic_events.json` 写入的事件字段的生成规则，便于后续调参时核对。

规则遵守 AIInfluence 侧 `Modules/AIInfluence/eventsgeneratorrules.txt` 的约束。

---

## 事件来源 Hook 一览

| 触发点 | 代码位置 | 事件语义 |
|--------|---------|---------|
| `ProxyWarBehavior.ApplyDiscoveryFallout` | `ProxyWarBridge.ProxyWarDiscoveryPatch` | 代理战争**阴谋败露**：公开的外交危机 |
| `ProxyWarBehavior.ApplyEspionageEffects` (玩家 AI 成功，未败露) | `ProxyWarBridge.ProxyWarAISuccessPatch` | **神秘事件**：目标国遭受不明破坏，幕后未知 |
| `IdeologyBehavior.TriggerGrandCoalition` | `FactionRebellionBridge.FactionGrandCoalitionPatch` | 意识形态派系组建反王**大联盟**（全面内战起点） |

所有事件最终走 `AIInfluenceWriter.WriteDynamicEvent(..., eventType, allowsDiplomaticResponse, expirationDays)` 写入本地 `bc_events_<saveFolder>.json`，用户在 MCM 里点 Sync 后才合并进 AIInfluence 的 `dynamic_events.json`。

## 字段生成规则

### 1. `importance`（重要程度 1-10）

**阴谋败露事件（Discovery）：**
- actionType 0-4：`MBRandom.RandomInt(2, 6)` → 随机 2-5
- actionType 5, 6, 8：固定 7
- actionType 7, 9：固定 8
- Grand Coalition：固定 9

**神秘事件（AISuccess，未败露的 AI 成功行动）：**
- actionType 0-4：`MBRandom.RandomInt(2, 5)` → 随机 2-4
- actionType 5, 6, 8：固定 5
- actionType 7, 9：固定 6

**分档理由：**
- 7/9 是**永久性政治重组**（雇佣军契约翻转、忠诚氏族倒戈），后果持续整场内战
- 5/6/8 是**一次性增益**（凝聚力/金钱/兵员），对长期格局影响较小
- 神秘事件整体比败露事件低 2 级（没人知道谁干的，外交冲击自然弱）

### 2. `type`（事件类型）

AIInfluence 定义的事件类型：`political` / `social` / `economic` / 其他。

| actionType | type |
|-----------:|------|
| 0 匪帮阻断商路 | `economic` |
| 1 城市暴动 | `social` |
| 2 宫廷丑闻 | `political` |
| 3 意识形态攻击 | `political` |
| 4 异见网络 | `social` |
| 5-9 代理战争叛军支援 | `political` |
| Grand Coalition | `political` |

Discovery 和 AISuccess 共用同一套 type 映射。

### 3. `allows_diplomatic_response`

按 `importance` 分档 roll：

| importance | p |
|-----------:|--:|
| ≤ 2 | 0.05 |
| 3 | 0.10 |
| 4 | 0.20 |
| 5 | 0.55 |
| 6 | 0.70 |
| 7 | 0.85 |
| ≥ 8 | 0.95 |

**分档理由**：对齐 AIInfluence `eventsgeneratorrules.txt` 的疾病规则——importance < 5 的局部/小规模事件"不得直接触发外交回应"，importance ≥ 5 的重大事件才能进入外交层面。importance 4 → 5 是关键跃迁点。

**神秘事件额外调整**：按 `importance - 2` 计算概率，体现"没人知道是谁干的，外交回应更稀有"（例如 AISuccess 的最高 importance=6 降到 4 档 → 概率 0.20）。

### 4. `expiration_campaign_days`

按 `importance` 线性插值到 10-84 天：

```
days = clamp(10 + (importance - 2) * 74 / 7, 10, 84)
```

| importance | expiration (天) |
|-----------:|---------------:|
| 2 | 10 |
| 3 | 21 |
| 4 | 31 |
| 5 | 42 |
| 6 | 52 |
| 7 | 63 |
| 8 | 73 |
| 9 | 84 |

即"小道谣言 10 天后没人再提，重大内战 84 天才会淡出"。

### 5. 固定字段

- `importance` 上限 10（AIInfluence 限制）
- `requires_diplomatic_analysis = true`（所有 BC 事件均需外交分析）
- `spread_speed = "normal"`
- `applicable_npcs = ["lords", "faction_leaders", "companions", "merchants"]`
- `player_involved = false`（BC 事件非玩家主导时；Discovery 里即使玩家参与也暂不标记）

## DeepSeek 扩写提示词约束

当启用 DeepSeek 时，`DeepSeekClient.CallApiAsync` 会发送事件摘要，要求模型输出 JSON `{ title, description }`：

- 必须保留所有人名、地名、数字、日期
- 日期标记 `【...】` / `[...]` 必须出现在 description 开头
- **标题要求**：中文四字或六字史诗标题（如"王冠蒙尘"、"旌旗易主"、"金流暗渡"），英文 2-4 词 epic title
- **禁止百分比（%）和数值比率**：改用叙事描述（"军心大振" 替代 "凝聚力+40"）
- **禁止现代管理学术语**：政治影响力 / 凝聚力 / 忠诚度 / 满意度 / 效率 → 威名 / 军心 / 民心 / 士气 / 朝望
- **王国统治者必须是叙事主体**。地方领主、氏族、大臣、祭司只能作为背景压力来源；不得作为外交主体
- 写作风格：古典史书笔法，加入环境描写 / 人物反应 / 政治影响分析
- 严禁编造新的人名或事实
- 失败时优雅降级：JSON 解析失败则把整段当 description，保留原标题

## 地理距离降级（Discovery 专用）

`ProxyWarBridge.ApplyGeographyDamping(ref importance, acting, target)` 基于两国最近一对定居点之间的 `GatePosition.Distance` 决定降级：

| 最近距离（地图单位） | 地理 tier | importance 调整 | 叙事前缀 |
|---------------------:|:---------:|:---------------:|---------|
| < 200 （相邻/接近） | 0 | ±0 | 无 |
| 200-400 （远邻） | 1 | −1 | 中："山川阻隔，风声辗转而至——" / EN: "Through distant lands the tidings came at last — " |
| ≥ 400 （跨大陆） | 2 | −2 | 中："海天相隔，此谋跨数国而来，令人难以尽信——" / EN: "Across half a continent this scheme is said to reach, scarce to be believed — " |

下限 importance = 2。卡拉迪亚东西长约 1000 单位，所以 400 单位相当于跨越大半个大陆。

**神秘事件不做地理降级**——因为 attribution 未知，对目标国造成的实际扰动与来源距离无关。
